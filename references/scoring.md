# Searching and Scoring

## How to search efficiently

Run **4–8 targeted searches**, not one broad one. Vary them:

- Combine role + modifier + portal: `AI automation engineer remote site:weworkremotely.com`
- Mix languages: search both `"AI engineer" remote LATAM` and `ingeniero IA remoto`
- Use the person's strongest angles as anchors (e.g., a rare stack combo like "Zoho + AI")
- Include recency terms only via the current year, not stale years

Aim for quality. Five to eight strong, real, current openings are more useful than twenty weak ones.

## Avoiding dead links and hallucinations

- Every opening must trace to an actual search result. Never invent a company, title, or URL.
- Prefer the direct posting URL. If only a listing/aggregator page is available, use that and note it.
- Postings close often — always remind the user to verify the link before applying.

## The fit score (1–5)

Score each opening honestly against the search profile:

- **5 — Bullseye.** Role, seniority, mode, and region all match, and the person's strongest angles are exactly what's asked for.
- **4 — Strong.** Clear match on role and mode; minor gaps (one preference off, or seniority slightly different).
- **3 — Plausible.** Right domain and mode, but notable gaps in stack, seniority, or region eligibility.
- **2 — Stretch.** Adjacent role or entry-level relative to the person; useful mainly as a market-floor reference.
- **1 — Weak.** Tangential; include only if the user wants breadth.

Do not inflate. A tracker where everything is a 5 is useless. Spread the scores to reflect real differences.

## What to capture per opening

- `titulo` / title
- `empresa` / company
- `ubicacion` / location + work mode
- `salario` / salary (blank if not shown — don't guess)
- `fit` (1–5)
- `fuente` / source portal
- `url` (direct link)
- `nota` — one sentence on why it fits (or the gap to watch)

## De-duplication

Before presenting, drop any opening the user already has in their tracker (match on title + company, case-insensitive).
